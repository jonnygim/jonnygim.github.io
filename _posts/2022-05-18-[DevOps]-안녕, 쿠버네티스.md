도커를 공부하다보니 매번 따라다니는 쿠버네티스를 한번 공부해봐야겠다고 생각했다.

도대체 쿠버네티스는 무엇인가?

## 쿠버네티스(Kubernetes)
쿠버네티스: 리눅스 컨테이너 작업을 자동화하는 오픈소스 플랫폼

- 리눅스 컨테이너 작업을 자동화하는 오픈 소스 플랫폼
- Google 엔지니어들이 개발하고 설계한 플랫폼이며 초창기에 리눅스 컨테이너 기술에 기여함.
- Google 은 쿠버네티스의 전신인 Borg를 통해 일주일에 20억 개 이상의 컨테이너를 생성하고 있었음
- Borg 를 개발하는 과정에서 축적된 경험이 쿠버네티스를 구성하는 기술의 주요 핵심이 됨.
- Google 이 2015에 새로 설립된 클라우드 네이티브 컴퓨팅 재단에 쿠버네티스를 기부함.

## 쿠버네티스의 주요 기능
- 여러 호스트에 걸쳐 컨테이너를 오케스트레이션 함.
- 하드웨어를 최대한 활용하여 애플리케이션을 실행하는 데 필요한 리소스를 극대화 함.
- 애플리케이션 배포 및 업데이트를 제어하고 자동화 함.
- 스토리지를 장착 및 추가해 stateful 애플리케이션을 실행 함.
- 컨테이너화된 애플리케이션과 해당 리소스를 즉시 확장함.
- 배포한 애플리케이션이 항상 배포 목적대로 실행되도록 함.
- 재동 배치, 재동 재시작, 자동 복제, 자동 확장을 사용해 애플리케이션 상태 확인과 셀프 복구를 수행함.

다른 오픈 소스 프로젝트를 함께 사용하면 좋음
- 레지스트리 : Atomic Registry 또는 Docker Registry
- 네트워킹 : OpenSwitch
- 텔레메트리: heapster, kibana, hawkular, elastic
- 보안 : Multi-tenancy 계층을 갖춘 LDAP, SELinux, RBAC, OAUTH
- 자동화 : Ansible
- 서비스 : 카탈로그

## 쿠버네티스 용어
- 마스터 : 쿠버네티스를 제어하는 머신. 모든 태스크 할당이 마스터에서 시작
- 노드 : 할댕된 태스크를 요청대로 수행하는 시스템. 마스터가 노드를 제어함
- 포드 : 
    - 단일 노드에 배포된 하나 이상의 컨테이너 그룹. 
    - 포드에 있는 모든 컨테이너는 IP 주소, IPC, 호스트 이름, 기타 리소스를 공유함. 
    - 기본 컨테이너에서 네트워크와 스토리지를 추상화 함
    - 이를 통해서 클러스터가 컨테이너를 더 쉽게 이동할 수 있음
- 복제 컨트롤러 : 클러스터에서 실행되어야 하는 동일한 포드 사본의 개수를 제어함
- 서비스 : 
    - 포드에서 작업 정의를 분리
    - 서비스 프록시는 클러스터에서 다른 위치로 이동한 경우나 교제된 경우, 서비스 요청을 적절한 포드로 자동 수신
- Kubelet : 
    - 노드에서 실행
    - 컨테이너 매니페스트를 읽고, 정의된 컨테이너가 시작되어 실행 중인지 확인
- kubectl : 명령줄 설정 툴

## 실행 프로세스
- 운영체제 기반으로 실행
- 노드에서 실행되는 컨테이너의 포드와 상호작용
- 마스터는 관리자로부터 명령을 전달받고 해당 명령을 하위 노드에 전달.
- 태스크에 가장 적합한 노드를 자동으로 결정
- 리소스를 할당하고 해당 노드에 포드를 지정하여 요청된 작업을 수행

컨테이너를 관리하는 방식에는 거의 변화가 없지만 컨테이너를 더 높은 수준에서 제어하기 때문에 개별 컨테이너나 노드를 세분화하여 관리하지 않고도 더 효과적으로 제어할 수 있음
대부분 쿠버네티스 마스터를 할당하고 노드와 포드를 정의하는 것.

## 도커와 함께
- 쿠버네티스가 노드에 대해 포드를 예약하면 해당 노드의 kubelet이 지정된 컨테이너를 실행하도록 Docker 에 명령
- kubelet 은 Docker 로부터 이러한 컨테이너의 상태를 계속해서 수집하고 마스터에서 해당 정보를 집계
- Docker 는 해당 노드에 컨테이너를 풀링하고 이러한 노드를 평소와 같이 시작하고 중지함
- 관리자가 모든 컨테이너의 모든 노드에서 작업을 직접 수행하는 것이 아니라 자동화된 시스템이 이러한 작업을 Docker에 요청
